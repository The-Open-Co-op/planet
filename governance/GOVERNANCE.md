# The Open Co-op — Governance

> Version 0.1 — March 2026
> Status: [DRAFT] — Open for feedback

This is a living document. It defines the minimum governance needed to operate effectively right now, with clear triggers for when and how it evolves.

## Purpose

The Open Co-op exists to build and steward cooperative digital infrastructure that serves its members and future generations. Our flagship project is PLANET — a Co-Operating System that combines a private app, a trust network, and a member-owned cooperative.

## Principles

1. **Member ownership** — The Open Co-op is owned by its members, not by shareholders or investors.
2. **Transparency** — Every financial transaction is public. Every strategic decision is documented.
3. **Future generations** — Inspired by the Well-being of Future Generations (Wales) Act 2015, we make decisions considering their impact on people not yet born. We build to last, not to exit.
4. **Progressive governance** — Our governance evolves as we grow. We start minimal and add structure when needed, not before.
5. **Practical cooperation** — We use the best tools for the job and prioritise getting things done over ideological purity.
6. **Trust as foundation** — We operate the way PLANET works: through real relationships, real accountability, and earned trust.

## Membership Tiers

| Tier | Commitment | Access | Voice |
|------|-----------|--------|-------|
| **Public** | None | Website, docs (public sections), Open Collective transparency | None |
| **Member** | Email + agree to principles | Community, docs (member sections), Loomio governance | Vote on strategic decisions |
| **Supporter** | Financial or equivalent support | All member access + working docs, deeper project visibility | Vote + input on feature priorities |
| **Contributor** | Active time/skill contribution | All supporter access + workstream participation, GitHub write access | Vote + voice in tactical decisions |
| **Core Team** | Stewarding the project | Full access including strategy and finance | Operational decision-making |

Membership is open to anyone who agrees to these principles. No one is excluded based on ability to pay — equivalent non-financial contributions are valued equally.

## Decision Making

### Right Now (Pre-launch, <100 members)

- **Operational decisions** (what to build this week, which tools to use, how to prioritise tasks): Core team decides, documents reasoning in GitHub.
- **Strategic decisions** (partnerships, major direction changes, governance changes, accepting grants): Proposed by core team, decided by membership vote on Loomio with minimum 7-day discussion period.
- **Financial decisions** (spending >£500): Core team proposes, membership informed via Open Collective transparency. Members can raise objections on Loomio.
- **Feature priorities**: Influenced by Open Collective bounty funding (members vote with contributions) and community input.

### Evolution Triggers

This governance is designed to evolve. Here's when:

| Milestone | Governance Change |
|-----------|------------------|
| **100 members** | Review this document. First formal governance vote. |
| **500 members** | Elect core team (rather than self-selected). Establish working groups with delegated authority. |
| **1,000 members** | Draft formal constitution. Consider incorporation as Community Benefit Society. |
| **5,000 members** | Full democratic governance. Elected board. Annual general meeting. |
| **10,000 members** | The Open Co-op is a mature cooperative. Governance should reflect that. |

Between milestones, any member can propose a governance change via Loomio at any time. Changes require a simple majority of participating voters with a minimum 14-day discussion period.

## Financial Model

All finances are transparent via Open Collective.

Revenue sources:
- Membership tiers (Supporter, Pioneer, Community)
- Feature bounties (members fund specific development)
- Grants (when aligned with our purpose)
- VTSP hosting services (post-launch)

Expenditure is approved by core team for amounts under £500 and by membership consultation for larger amounts. All transactions are visible on Open Collective.

## Conflict Resolution

1. Discuss directly with the person involved
2. If unresolved, bring to core team for mediation
3. If still unresolved, bring to Loomio for community input
4. The community's decision is final

## Amendments

This document can be amended by:
- Core team for minor clarifications (documented in Git history)
- Loomio vote for substantive changes (minimum 7-day discussion, simple majority)

---

*This governance is intentionally minimal. We'd rather have a one-page document that everyone reads and follows than a fifty-page constitution that nobody reads. Structure will grow as the community grows.*
