# Contributing to PLANET

Thank you for your interest in building PLANET. This is a cooperative project — every contribution matters.

## Ways to Contribute

**Anyone can:**
- Report issues or suggest improvements
- Share PLANET with people who'd care about it
- Fund features via [Open Collective](https://opencollective.com/the-open-co-op)

**Members can:**
- Participate in governance decisions on Loomio
- Provide feedback on specs and designs
- Test demos and prototypes

**Contributors can:**
- Write or improve documentation
- Design UX flows and interfaces
- Write code
- Create content (blog posts, explainers, videos)
- Help with community organising and outreach

## How We Work

### Docs and Specs
- Working drafts live in Google Docs (ask for access in the contributor Signal group)
- Stable docs get promoted to the [docs site](https://docs.open.coop) via PRs to the `docs` repo
- Specs and architecture decisions live in this repo under `specs/` and `adr/`

### Code
- All code goes through pull requests with at least one review
- Write clear commit messages
- If you're unsure about approach, open an issue to discuss before building

### Communication
- **Signal groups** for real-time coordination (contributor access)
- **GitHub Issues** for task tracking and technical discussion
- **Loomio** for governance decisions (member access)
- **Open Collective** for financial transparency (public)

### Using AI
We encourage using AI tools (Claude, etc.) to accelerate your work. When you do:
- Always review AI output before committing — you own the quality
- Note in PRs if substantial portions were AI-assisted
- Share useful prompts or approaches with other contributors

## Decision Making

At this stage (pre-launch), the core team makes operational decisions. Strategic decisions go to the membership via Loomio. This governance evolves as the community grows — see [GOVERNANCE.md](governance/GOVERNANCE.md).

## Code of Conduct

Be kind. Be honest. Assume good faith. Remember we're building infrastructure for future generations — act accordingly.

Specific expectations:
- Treat all contributors with respect regardless of experience level
- Give constructive feedback, not personal criticism
- Keep discussions focused and productive
- Respect the time people are volunteering
- If you disagree, propose an alternative rather than just objecting

## Getting Started

1. Read the [docs site](https://docs.open.coop) to understand the vision
2. Browse [open issues](https://github.com/The-Open-Co-op/planet/issues) for something that interests you
3. Comment on an issue to signal you'd like to work on it
4. Ask questions — there are no stupid questions in a cooperative
